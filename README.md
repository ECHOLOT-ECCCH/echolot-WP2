# Overview

Work Package 2 (**WP2**) of the ECHOLOT project focuses on defining the overall architecture and technical specifications of the ECHOLOT system to ensure its seamless integration with the European Collaborative Cloud for Cultural Heritage (ECCCH), Data Space for Cultural Heritage (DS4CH), and other platforms like Europeana and the Wikimedia ecosystem.
This repository contains the deliverables, documentation, and technical artifacts produced by WP2, including:

- System requirements and architecture designs.
- Interoperability models and mappings.
- Provenance and rights metadata specifications.
- MediaWiki plug-ins and extensions.

# System architecture 

ECHOLOT operates as an interoperability hub within a complex ecosystem of external actors and systems. The C4 model is used to represent the architecture at different levels of abstraction:

- Context (Level 1): High-level view of ECHOLOT in relation to its users and external systems.
- Containers (Level 2): High-level technical building blocks (e.g., ECHOLOT Core, ECHOLOT Enrich).
- Components (Level 3): Internal structure of containers (e.g., Backend, Frontend, Workflow Engine).
- Code (Level 4): Optional layer and not represented here

The draft deliverable of the system specification is available in this repository and will be continuously updated.
